# Codex Brainstorm 模板

## Phase 1: Claude 獨立分析

```markdown
## Claude 獨立分析

### 問題本質

#### 表面需求

用戶說要 [X]

#### 5 Why 追問

1. Why: 為什麼要 [X]？→ 因為 [Y]
2. Why: 為什麼 [Y] 重要？→ 因為 [Z]
3. Why: 為什麼 [Z] 是問題？→ 因為 [...]
4. Why: ...
5. Why: ...

#### 核心問題

真正要解決的是 [核心問題]

### 約束盤點

| 類型 | 約束 | 彈性     | 如果打破... |
| ---- | ---- | -------- | ----------- |
| 技術 | ...  | 無/低/中 | 會怎樣？    |
| 業務 | ...  | ...      | ...         |
| 資源 | ...  | ...      | ...         |

### Claude 的最優解假設

**方案：[具體方案]**

理由：

1. [為什麼這是最優的]
2. [...]

自我批判（潛在弱點）：

1. [弱點 1]
2. [弱點 2]

### 準備對抗

預設 Codex 可能提出的方案：

1. [方案 A] → 我的攻擊點：[...]
2. [方案 B] → 我的攻擊點：[...]
```

## Phase 3: 辯論記錄

```markdown
## 辯論記錄

### Round N

**Codex 立場**：[方案 Y]

- 理由：[...]
- 自認弱點：[...]

**Claude 攻擊**：

1. [攻擊點 1]
2. [攻擊點 2]

**Codex 反駁**：

1. [反駁 1]
2. [反駁 2]

**Claude 回應**：

- 對於反駁 1：[同意/不同意] + [理由]
- 對於反駁 2：[同意/不同意] + [理由]

**立場更新**：

- Claude：[維持/更新為 Z]
- Codex：[維持/更新為 W]

**均衡檢查**：

- Claude 還有新攻擊？[是/否]
- Codex 還有新攻擊？[是/否]
```
