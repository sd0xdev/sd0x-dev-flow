# 納什均衡判定

## 定義

```
納什均衡 = 給定對方的策略，任何一方都無法通過單方面改變策略來改善自己的結果

技術方案的均衡 = 給定所有約束和反駁，任何一方都無法提出新的有效攻擊來推翻當前共識
```

## 判定流程

```
┌─────────────────────────────────────────┐
│ 輪次 N 結束                              │
├─────────────────────────────────────────┤
│ Q1: Claude 能提出新攻擊嗎？              │
│     - 是 → 繼續辯論                      │
│     - 否 → Q2                            │
├─────────────────────────────────────────┤
│ Q2: Codex 能提出新攻擊嗎？               │
│     - 是 → 繼續辯論                      │
│     - 否 → 達到均衡                      │
└─────────────────────────────────────────┘
```

## 均衡類型

| 類型             | 定義                           | 報告方式                            |
| ---------------- | ------------------------------ | ----------------------------------- |
| **純策略均衡**   | 雙方收斂到同一方案             | 「均衡解：方案 X」                  |
| **混合策略均衡** | 雙方同意在不同條件下用不同方案 | 「條件均衡：若 A 則 X，若 B 則 Y」  |
| **Pareto 最優**  | 存在多個均衡，無法比較優劣     | 「Pareto 集：{X, Y, Z}」            |
| **無均衡**       | 達到最大輪數仍有分歧           | 「分歧點：[列出]，需要 [額外資訊]」 |

## 每輪均衡檢查

```markdown
## 均衡檢查 (輪次 N)

### 我的立場

- 當前方案：[X]
- 是否更新：[是/否]

### 攻擊能力檢查

- 我還能對 Codex 的方案提出新的、未被反駁的攻擊嗎？
  - [ ] 是 → 繼續（列出攻擊）
  - [ ] 否 → 我方達到均衡

### Codex 攻擊能力檢查

- Codex 還能對我的方案提出新的、未被反駁的攻擊嗎？
  - [ ] 是 → 繼續
  - [ ] 否 → 雙方達到均衡

### 均衡狀態

- [ ] 繼續辯論
- [ ] 達到純策略均衡：方案 [X]
- [ ] 達到條件均衡：若 [A] 則 [X]，若 [B] 則 [Y]
- [ ] Pareto 最優集：{X, Y}
- [ ] 最大輪數，輸出分歧報告
```

## 無法達成均衡時的處理

```markdown
# 分歧報告：[主題]

## 辯論摘要

- 總輪數：N（達到上限）
- 仍有分歧

## 分歧點

| 議題 | Claude 立場 | Codex 立場 | 分歧原因 |
| ---- | ----------- | ---------- | -------- |
| [X]  | ...         | ...        | 假設不同 |

## 需要額外資訊

1. [如果知道 A，可以判斷 B]
2. [如果確認 C 約束，可以收斂]

## 條件建議

- 如果 [條件 1]：建議 Claude 方案
- 如果 [條件 2]：建議 Codex 方案
```
