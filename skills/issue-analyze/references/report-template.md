# Issue 分析報告模板

## 標準報告

```markdown
# Issue #<number> 分析報告

## Issue 摘要

| 欄位     | 內容         |
| -------- | ------------ |
| 標題     | <title>      |
| 標籤     | <labels>     |
| 報告者   | <reporter>   |
| 報告時間 | <created_at> |
| 症狀     | <症狀描述>   |

## 問題分類

| 維度   | 判斷結果                          |
| ------ | --------------------------------- |
| 時間性 | 回歸 / 一直如此 / 不確定          |
| 確定性 | 必現 / 間歇性                     |
| 複雜度 | 低 / 中 / 高                      |
| 類型   | 邏輯錯誤 / 效能 / 資料問題 / 其他 |

**選擇策略**：`/code-explore` | `/git-investigate` | `/code-investigate` | `/codex-brainstorm`

## 調查過程

### Phase 1: <策略名稱>

<調查過程摘要>

### Phase 2: <如有第二階段>

<調查過程摘要>

## 根因分析

### 直接原因

<導致問題的直接原因>

### 根本原因

<為什麼會發生這個直接原因>

### 相關代碼

| 檔案      | 行號 | 說明       |
| --------- | ---- | ---------- |
| `src/...` | L123 | <問題代碼> |

## 修復建議

### 建議方案

<修復方案描述>

### 預估影響

- 影響範圍：<範圍>
- 風險等級：低 / 中 / 高
- 需要測試：Unit / Integration / E2E

## 後續行動

- [ ] 修復代碼
- [ ] 補充測試
- [ ] 更新文檔（如需）
```

## 快速報告（簡化版）

```markdown
# Issue #<number> 快速分析

**症狀**：<一句話描述>

**分類**：<類型> → 策略：<策略>

**根因**：<一句話描述根因>

**修復**：<修復建議>

**相關檔案**：`src/xxx.ts:123`
```

## 分歧報告（用於 /codex-brainstorm 結果）

```markdown
## Claude 視角

- 根因推測：<Claude 的結論>

## Codex 視角

- 根因推測：<Codex 的結論>
- 依據：<論據>

## 共識點

- <雙方同意的結論>

## 分歧點

| 議題     | Claude | Codex  |
| -------- | ------ | ------ |
| <議題 1> | <立場> | <立場> |

## 最終結論

<綜合分析後的結論>
```
